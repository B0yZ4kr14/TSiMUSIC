⚠️ Você está acessando via endereço IP.
Por isso o navegador exibe um aviso de segurança. Abaixo explicamos o motivo e como resolver.

🔒 Por que aparece o aviso?

O acesso pelo endereço IP (100.86.64.1:8443) utiliza um certificado autoassinado. O navegador não reconhece a autoridade que emitiu esse certificado, então mostra um aviso de segurança.

Isso é normal e esperado para acesso interno via IP. Os dados continuam criptografados (HTTPS).

🚀 Acesso recomendado (sem avisos)

Use o hostname oficial do servidor Tailscale. Ele possui um certificado Let's Encrypt válido, reconhecido por todos os navegadores.

🛠️ Como instalar o certificado no sistema

Se precisar acessar regularmente pelo IP, você pode instalar o certificado autoassinado no trust store do seu sistema operacional.

Opção A — Script automático:

curl -fsSL https://midiaserver-saudeclinica.tailbda57.ts.net/install-cert.sh | bash

Opção B — Manual:

# 1. Baixe o certificado
openssl s_client -connect 100.86.64.1:8443 -showcerts </dev/null 2>/dev/null | \
  awk '/BEGIN CERTIFICATE/,/END CERTIFICATE/{print}' > ip-tailscale.crt

# 2. Copie para o diretório de CAs
sudo cp ip-tailscale.crt /usr/local/share/ca-certificates/

# 3. Atualize o trust store
sudo update-ca-certificates

sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain ip-tailscale.crt

Passo a passo manual:

  1. Baixe o certificado: ip-tailscale.crt
  2. Clique duplo no arquivo ip-tailscale.crt
  3. Clique em Instalar certificado…
  4. Selecione Máquina local → Avançar
  5. Escolha Colocar todos os certificados no repositório a seguir
  6. Selecione Autoridades de certificação raiz confiáveis
  7. Conclua o assistente e reinicie o navegador
✅ Após a instalação: Feche e reabra o navegador. O aviso deverá desaparecer ao acessar https://100.86.64.1:8443/.
